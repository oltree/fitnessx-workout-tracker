# FitnessX - Трекер тренировок

> [English version / Английская версия](README.md)

**FitnessX** - современное фитнес-приложение для создания, планирования и отслеживания персональных тренировок. Разработано полностью на клиентской стороне (React + TypeScript), без серверной части.

## Обзор

FitnessX позволяет пользователям регистрироваться, входить в аккаунт, планировать тренировки в интерактивном календаре, добавлять упражнения с настраиваемыми параметрами и отслеживать прогресс выполнения - всё сохраняется в `localStorage` браузера через Redux-Persist, без необходимости в бэкенде.

Приложение также интегрировано с **OpenWeatherMap API** - при планировании тренировок на ноги оно анализирует текущую погоду и выдаёт рекомендации.

## Функциональность

| Функция | Описание |
|---|---|
| **Аутентификация** | Регистрация и вход по email/паролю, данные хранятся в localStorage |
| **Главная страница** | Баннер с калькулятором ИМТ, форма создания тренировки, мини-календарь |
| **Конструктор тренировки** | Многоэтапная форма: выбор даты → список упражнений → название тренировки |
| **Параметры упражнений** | Подходы, повторения, длительность, вес - настраивается для каждого упражнения |
| **Интерактивный календарь** | Виды месяц/неделя/день, перетаскивание тренировок для переноса |
| **Страница тренировки** | Просмотр упражнений, отметка выполнения каждого |
| **Лента уведомлений** | События входа, регистрации, погодные предупреждения |
| **Профиль** | Информация о пользователе, история тренировок с деталями, выход |
| **Интеграция с погодой** | Проверка погоды (Минск/Москва) при добавлении упражнений на ноги |

## Управление состоянием

Приложение использует **Redux Toolkit** с **Redux-Persist**. Состояние разделено на пять слайсов:

| Слайс | Сохраняется | Содержимое |
|---|---|---|
| `user` | Да | `email`, `firstName`, `lastName`, `isAuth` |
| `workouts` | Да | Полный список `Workout[]`, отсортированный по дате |
| `exercises` | Нет | Временный список упражнений при создании тренировки |
| `notifications` | Нет | Записи уведомлений |
| `workout` (current) | Нет | Текущая открытая тренировка для страницы деталей |

В `localStorage` сохраняются только `user` и `workouts` - сессионное состояние живёт исключительно в Redux.

Пользовательские хуки (`useWorkouts`, `useUser` и др.) оборачивают мемоизированные селекторы, отделяя компоненты от Redux API.

## Маршрутизация и защита роутов

| Роут | Компонент | Требует авторизации |
|---|---|---|
| `/` | Welcome | Нет |
| `/signup` | SignUp | Нет |
| `/signin` | SignIn | Нет |
| `/home` | Home | Да |
| `/workout` | Workout | Да |
| `/notifications` | Notifications | Да |
| `/profile` | Profile | Да |
| `*` | NotFound | - |

Компонент `Router.tsx` проверяет флаг `isAuth` из Redux и условно рендерит защищённые или публичные роуты. Неавторизованные пользователи перенаправляются на `/`.

## API-интеграции

### Сервис погоды (`src/services/weather.service.ts`)

Класс-синглтон для работы с **OpenWeatherMap API**:
- `getWeatherData(city)` - получает текущую погоду для Минска или Москвы
- `isBadWeather(activity, data)` - возвращает `true` для упражнений на ноги при: дожде **или** температуре ниже 10°C
- `getWeatherMessage(city, activity)` - возвращает строку с рекомендацией для пользователя

Проверка погоды запускается автоматически, когда пользователь добавляет упражнение типа **ноги**, и создаёт уведомление с результатом.

### Аутентификация (localStorage)

Бэкенд API отсутствует. Пользователи хранятся в виде JSON в `localStorage['users']`. При входе приложение ищет запись по email и проверяет пароль, затем записывает состояние авторизации в Redux (сохраняется через Redux-Persist).

## Запуск проекта

### Требования
- Node.js 16+
- npm или yarn

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/oltree/fitnessx.git
cd fitnessx

# Установить зависимости
npm install

# Запустить сервер разработки
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000`.

### Доступные скрипты

| Скрипт | Описание |
|---|---|
| `npm run dev` | Запустить dev-сервер через Craco |
| `npm run build` | Продакшн-сборка |
| `npm test` | Запустить тесты |
| `npm run deploy` | Собрать и задеплоить на GitHub Pages |
